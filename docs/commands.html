DevDoctor docs

Commands

Each diagnostic command is read-only and supports --path, --format, --ci, --strict, --only, --summary-only, and --no-hints where applicable.

CommandPurposeExample
envCheck dotenv files and configured env rules.php devdoctor env --strict
cacheCheck framework and tool cache directories, size, permissions, and Laravel artifacts.php devdoctor cache --max-size=512
httpCheck configured HTTP URLs without making network requests.php devdoctor http --url=https://example.test
portsCheck local development port conflicts.php devdoctor ports --common
phpCheck PHP runtime, composer platform requirements, memory, php.ini, and Xdebug in CI.php devdoctor php --ci
nodeCheck Node.js project and package manager health.php devdoctor node
frontendCheck frontend presets and build readiness for Vite, Next.js, Nuxt, Astro, and generic frontend projects.php devdoctor frontend
flutterCheck Flutter and Dart pubspec, lockfiles, SDK constraints, dependency sources, and platform markers.php devdoctor flutter
mobileCheck native Android and iOS project markers, wrappers, debug flags, and lockfiles.php devdoctor mobile
monorepoCheck monorepo tooling, workspace lockfiles, and root package scripts.php devdoctor monorepo
pythonCheck Python manifests and dependency manager health for pip, Poetry, Pipenv, uv, and Conda.php devdoctor python
rubyCheck Ruby and Rails manifests, lockfiles, versions, credentials, database config, and gem sources.php devdoctor ruby
goCheck Go modules, sums, workspaces, local replace directives, toolchain directives, and vendor metadata.php devdoctor go
rustCheck Cargo manifests, lockfiles, workspaces, toolchains, dependency sources, and release profiles.php devdoctor rust
javaCheck Maven, Gradle, Ant, wrappers, Java versions, risky build scripts, and Spring profile red flags.php devdoctor java
iacCheck Terraform, OpenTofu, and Terragrunt manifests, lockfiles, module refs, and secret hygiene.php devdoctor iac
kubeCheck Kubernetes manifests and Helm charts, locks, images, services, and secret hygiene.php devdoctor kube
dotnetCheck .NET solutions, projects, SDK pinning, target frameworks, lock mode, and NuGet sources.php devdoctor dotnet
cppCheck C/C++ build files, dependency managers, compile commands, and portability risks.php devdoctor cpp
webCheck generic web entry files, asset references, public config, server hints, and port declarations.php devdoctor web
laravelCheck Laravel .env, APP_KEY, debug mode, APP_URL, runtime directories, and config cache.php devdoctor laravel
symfonyCheck Symfony .env hygiene, APP_SECRET, runtime directories, recipe drift, and Composer scripts.php devdoctor symfony
securityCheck env examples, hard-coded secrets, risky scripts, and risky Compose settings.php devdoctor security
composerCheck Composer project health without install/update.php devdoctor composer
depsRun Composer and Node dependency diagnostics together.php devdoctor deps --summary-only
dbCheck database environment configuration, with optional read-only PDO connection checks.php devdoctor db --connect
queueCheck queue environment configuration and production sync risks.php devdoctor queue --strict
gitCheck repository hygiene and sensitive files.php devdoctor git --require-clean
dockerCheck Compose files, daemon state, ports, and containers.php devdoctor docker --compose-file=docker-compose.yml
healthRun a broad local project health check, with ports available through --include-ports.php devdoctor health --format=json
doctorAlias for health with the same broad local project checks.php devdoctor doctor --summary-only
ciRun CI-safe diagnostics and aggregate exit codes.php devdoctor ci --modules=env,php,node,laravel,composer,db,git,docker
presetsDetect supported project presets, including PHP, Node/frontend, Flutter/Dart, native mobile, monorepos, Python, Go, Rust, JVM, IaC, Kubernetes/Helm, C/C++, .NET, generic web, and Docker Compose evidence.php devdoctor presets --format=json
inventoryShow detected presets, available modules, and auto-selected modules.php devdoctor inventory --format=json
commandsList DevDoctor commands and their documentation metadata.php devdoctor commands --format=json
explainExplain issue codes and their built-in hints, optionally filtered by module.php devdoctor explain --module=env --format=json
policyShow DevDoctor safety and compatibility policy.php devdoctor policy --format=json
support-bundlePrint redacted support context without writing files.php devdoctor support-bundle
initPreview and optionally write devdoctor.yml.php devdoctor init --dry-run

v1.29.0 adds utility commands: inventory, explain, policy, and support-bundle. They expose project inventory, issue-code hints, safety policy, and redacted support context without modifying project files.

v1.33.0 adds commands and explain --module=... for terminal-native discovery of command metadata and issue-code hints.

Machine-readable reference

Command metadata ships with the documentation.

The static docs include docs/commands.json for tooling that wants command names, modules, read-only status, examples, and high-level categories without scraping HTML.
