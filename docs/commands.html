Commands

Each diagnostic command is read-only and supports `--path`, `--format`, `--ci`, and `--strict` where applicable.

HomeConfigOutput Formats
CommandPurposeExample
envCheck dotenv files and configured env rules.php devdoctor env --strict
portsCheck local development port conflicts.php devdoctor ports --common
phpCheck PHP runtime, composer platform requirements, memory, php.ini, and Xdebug in CI.php devdoctor php --ci
nodeCheck Node.js project and package manager health.php devdoctor node
composerCheck Composer project health without install/update.php devdoctor composer
gitCheck repository hygiene and sensitive files.php devdoctor git --require-clean
dockerCheck Compose files, daemon state, ports, and containers.php devdoctor docker --compose-file=docker-compose.yml
ciRun CI-safe diagnostics and aggregate exit codes.php devdoctor ci --modules=env,php,node,composer,git,docker
presetsDetect supported project presets.php devdoctor presets --format=json
initPreview and optionally write `devdoctor.yml`.php devdoctor init --dry-run
