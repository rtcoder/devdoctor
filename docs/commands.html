Commands

Each diagnostic command is read-only and supports `--path`, `--format`, `--ci`, `--strict`, `--only`, `--summary-only`, and `--no-hints` where applicable.

HomeConfigOutput Formats
CommandPurposeExample
envCheck dotenv files and configured env rules.php devdoctor env --strict
cacheCheck framework and tool cache directories, size, permissions, and Laravel artifacts.php devdoctor cache --max-size=512
httpCheck configured HTTP URLs without making network requests.php devdoctor http --url=https://example.test
portsCheck local development port conflicts.php devdoctor ports --common
phpCheck PHP runtime, composer platform requirements, memory, php.ini, and Xdebug in CI.php devdoctor php --ci
nodeCheck Node.js project and package manager health.php devdoctor node
frontendCheck frontend presets and build readiness for Vite, Next.js, Nuxt, Astro, and generic frontend projects.php devdoctor frontend
laravelCheck Laravel .env, APP_KEY, debug mode, APP_URL, runtime directories, and config cache.php devdoctor laravel
securityCheck env examples, hard-coded secrets, risky scripts, and risky Compose settings.php devdoctor security
composerCheck Composer project health without install/update.php devdoctor composer
depsRun Composer and Node dependency diagnostics together.php devdoctor deps --summary-only
dbCheck database environment configuration, with optional read-only PDO connection checks.php devdoctor db --connect
queueCheck queue environment configuration and production sync risks.php devdoctor queue --strict
gitCheck repository hygiene and sensitive files.php devdoctor git --require-clean
dockerCheck Compose files, daemon state, ports, and containers.php devdoctor docker --compose-file=docker-compose.yml
healthRun a broad local project health check, with ports available through --include-ports.php devdoctor health --format=json
doctorAlias for health with the same broad local project checks.php devdoctor doctor --summary-only
ciRun CI-safe diagnostics and aggregate exit codes.php devdoctor ci --modules=env,php,node,laravel,composer,db,git,docker
presetsDetect supported project presets, including PHP, Node/frontend, Python, Go, Rust, JVM, C/C++, .NET, generic web, and Docker Compose evidence.php devdoctor presets --format=json
initPreview and optionally write `devdoctor.yml`.php devdoctor init --dry-run

v1.14.0 adds frontend diagnostics and expands Node package manager checks for npm, Yarn, pnpm, and Bun. Health and CI add frontend automatically when frontend presets are detected.
